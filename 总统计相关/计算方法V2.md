# 总统计页面配置文档

## 一、通用数据处理规则

### 1. SQL查询规则
- ✅ 允许：`SUM(CASE WHEN MONTH(date)=1 THEN sum_elec...)` - 按月汇总
- ❌ 禁止：`SUM(sum_elec) as sum_elec` - 不在SQL中累加年度合计

### 2. JavaScript处理规则（重要！）
```javascript
// 年度合计：先四舍五入每月值，再累加
row.sum_elec = 0;
for (let m = 1; m <= 12; m++) {
    const monthVal = row[`month${m}_elec`] || 0;
    if (monthVal > 0) {
        row.sum_elec += Math.round(monthVal);  // 先四舍五入，再累加
    }
}

// 合计行：累加四舍五入后的值
const roundedValue = Math.round(value);
totals[cat] += roundedValue;  // ✅ 正确
```

### 3. 百分比计算
```javascript
percentage = (roundedValue / roundedTotal * 100).toFixed(2)
```

## 三、数据来源说明

### 1. 系统采集数据
- 历史数据：`sum_elec_group_by_day`（按日期范围查询）
- 今日实时数据：`data_all`（当查询日期包含今天且时间>=7:00时）
  - 判断条件：`HOUR(NOW()) >= 7`
  - 取数时间：`CURDATE() + 7小时` 或 `前一天 + 7小时`
  - 计算方式：`MAX(KWH) - MIN(KWH)`，按NO分组
  - 过滤条件：`KWH > 0 AND KWH < 9999999999`

### 2. 人工录入数据
- 数据表：`data_elec_temp_input`
- 字段：
  - `sum_elec`：电量（度）
  - `production`：产量或其他数值（如就餐人数、水单价）

### 3. 公司总用电量
- 数据表：`data_elec_temp_input`
- NO：10000（人工录入）
- 用途：作为占比计算的分母

## 四、显示规则

1. 数值为0或null时，表格显示空白（不显示"0"或"0%"）
2. 电量值保留整数，千分位格式化
3. 占比保留2位小数，带%符号
4. 人工录入数据在表格中标红显示
5. 单台计算保留2位小数（使用原始值，不四舍五入）

## 五、数据一致性检查清单

- [ ] SQL查询中没有 `SUM(sum_elec) as sum_elec`
- [ ] JavaScript中为每行计算 `sum_elec`（先四舍五入再累加）
- [ ] 合计行计算也遵循先四舍五入再累加
- [ ] 表格显示使用 `Math.round()` 四舍五入
- [ ] 百分比计算使用四舍五入后的值
- [ ] 图表数据使用四舍五入后的值
- [ ] 饼图百分比手动计算，不依赖 ECharts 自动计算
