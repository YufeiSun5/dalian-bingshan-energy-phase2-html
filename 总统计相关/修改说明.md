## 修改的问题及解决方案

### 1. 图表中移除汇总项（工厂管理本部）
**问题：** "工厂管理本部"是其他5个工厂部门的求和，在图表中显示会造成数据重复计算。

**解决方案：**
在 `updateCharts` 函数中，修改图表显示的部门列表：
```javascript
// 修改前
const chartDepts = ['factory_total', 'machining', 'motor', ...];

// 修改后（移除 factory_total）
const chartDepts = ['machining', 'motor', 'assembly1', 'assembly2', 'power_facility', 
                   'dev_dept', 'planning_dept', 'quality_dept', 'sales_dept', 'loss'];
```

### 2. 移除饼图扇区之间的分割线
**问题：** 饼图各扇区之间有明显的分割线，影响美观。

**解决方案：**
修改饼图配置的 `itemStyle`：
```javascript
// 修改前
itemStyle: { borderWidth: 5, borderColor: '#1d386f' }

// 修改后
itemStyle: { borderWidth: 0, borderColor: 'transparent' }
```

### 3. 表格与饼图百分比不一致（精度问题）
**问题：** 表格显示的百分比和饼图显示的百分比有0.01%的差异。

**原因分析：**
- 表格显示的是四舍五入后的整数值
- 但百分比计算时使用的是原始小数值
- 饼图使用ECharts自动计算的百分比

**解决方案分三步：**

#### 步骤1：表格数据处理 - 统一使用四舍五入后的值计算百分比
```javascript
// 在 transformDataForLayui 函数中
const value = values[key] || 0;
const roundedValue = Math.round(value);  // 先四舍五入
const companyTotal = values['total'] || 0;
const roundedCompanyTotal = Math.round(companyTotal);

// 显示数值
row[`${key}_val`] = roundedValue > 0 ? roundedValue.toLocaleString() : '';

// 计算百分比 - 使用四舍五入后的值
row[`${key}_pct`] = (roundedCompanyTotal > 0 && roundedValue > 0) 
    ? `${(roundedValue / roundedCompanyTotal * 100).toFixed(2)}%` 
    : '';
```

合计行同样处理：
```javascript
const totalRow = { month: '合计', isClickable: true, isTotal: true };
const grandTotal = totals['total'];
const roundedGrandTotal = Math.round(grandTotal);
categories.forEach(key => {
    const value = totals[key];
    const roundedValue = Math.round(value);
    totalRow[`${key}_val`] = roundedValue > 0 ? roundedValue.toLocaleString() : '';
    totalRow[`${key}_pct`] = (roundedGrandTotal > 0 && roundedValue > 0) 
        ? `${(roundedValue / roundedGrandTotal * 100).toFixed(2)}%` 
        : '';
});
```

#### 步骤2：饼图数据源 - 使用四舍五入后的值
```javascript
// 修改前
const pieChartData = chartDepts.map(key => ({ 
    name: departmentConfig[key].title, 
    value: values[key] || 0 
}));

// 修改后
const pieChartData = chartDepts.map(key => ({ 
    name: departmentConfig[key].title, 
    value: Math.round(values[key] || 0)  // 四舍五入
}));
```

#### 步骤3：饼图百分比显示 - 手动计算百分比
不使用ECharts自动计算的百分比，改为手动计算，确保与表格算法一致：

```javascript
series: [{
    // ... 其他配置
    label: { 
        formatter: function(params) {
            // 手动计算百分比，确保与表格一致
            const total = pieChartData.reduce((sum, item) => sum + item.value, 0);
            const percentage = total > 0 ? ((params.value / total * 100).toFixed(2)) : '0.00';
            return `${percentage}%`;
        }, 
        // ... 其他样式
    },
    // tooltip 也同样处理
}]
```

完整的 tooltip formatter：
```javascript
tooltip: { 
    trigger: 'item', 
    formatter: function(params) {
        const total = pieChartData.reduce((sum, item) => sum + item.value, 0);
        const percentage = total > 0 ? ((params.value / total * 100).toFixed(2)) : '0.00';
        return `${params.seriesName}<br/>${params.name}: ${params.value.toLocaleString()} (${percentage}%)`;
    }
}
```

### 4. 饼图初始化时无法加载（动力实施课页面）

**问题：** 在修复百分比不一致问题后，饼图无法显示，控制台报错。

**原因分析：**
在饼图初始化时（`initChart`函数），配置了 `label.formatter` 和 `tooltip.formatter` 函数，这些函数试图访问 `params.data.pieDataRef` 属性来计算百分比总值。但在初始化阶段，`data: []` 为空数组，不存在 `pieDataRef` 属性，导致：
- `params.data.pieDataRef.reduce()` 调用失败
- JavaScript报错：Cannot read property 'reduce' of undefined
- 饼图无法渲染

**错误代码示例：**
```javascript
// 初始化时的错误配置
pieChart.setOption({
    series: [{
        data: [],  // 空数据
        label: { 
            formatter: function(params) {
                // ❌ 此时 params.data 为空，pieDataRef 不存在
                const total = params.data.pieDataRef.reduce(...);  // 报错！
                return `${percentage}%`;
            }
        }
    }]
});
```

**解决方案：**

将 formatter 配置从初始化阶段移除，只在数据更新时（`updateChart` 和 `updatePieChartForMonth` 函数中）设置完整配置。

#### 修改1：简化初始化配置
```javascript
// initChart 函数中
pieChart.setOption({
    backgroundColor: 'transparent',
    tooltip: {
        trigger: 'item',
        backgroundColor: '#1d386f',
        borderColor: '#2a6dc8',
        textStyle: { color: '#fff' }
        // ✅ 移除 formatter，在数据更新时再设置
    },
    legend: { /* ... */ },
    series: [{
        name: '用电占比',
        type: 'pie',
        data: [],
        itemStyle: { borderWidth: 2, borderColor: '#1d386f' },
        // ✅ 移除 label.formatter，在数据更新时再设置
        labelLine: { lineStyle: { color: 'rgba(255, 255, 255, 0.5)' } }
    }]
});
```

#### 修改2：在数据更新时设置完整配置
```javascript
// updateChart 函数中
function updateChart() {
    // ... 构建 pieData 数组
    
    // ✅ 在有数据时才设置 formatter
    pieChart.setOption({
        tooltip: {
            formatter: function(params) {
                const total = pieData.reduce((sum, item) => sum + item.value, 0);
                const percentage = total > 0 ? ((params.value / total * 100).toFixed(2)) : '0.00';
                return `${params.seriesName}<br/>${params.name}: ${params.value.toLocaleString()} (${percentage}%)`;
            }
        },
        series: [{
            data: pieData,
            label: { 
                formatter: function(params) {
                    const total = pieData.reduce((sum, item) => sum + item.value, 0);
                    const percentage = total > 0 ? ((params.value / total * 100).toFixed(2)) : '0.00';
                    return `${percentage}%`;
                }, 
                color: '#fff', 
                position: 'outside' 
            }
        }]
    });
}
```

#### 修改3：updatePieChartForMonth 同样处理
```javascript
// updatePieChartForMonth 函数中
function updatePieChartForMonth(month) {
    // ... 构建 pieData 数组
    
    pieChart.setOption({
        tooltip: {
            formatter: function(params) {
                const total = pieData.reduce((sum, item) => sum + item.value, 0);
                const percentage = total > 0 ? ((params.value / total * 100).toFixed(2)) : '0.00';
                return `${params.seriesName}<br/>${params.name}: ${params.value.toLocaleString()} (${percentage}%)`;
            }
        },
        series: [{
            data: pieData,
            label: { 
                formatter: function(params) {
                    const total = pieData.reduce((sum, item) => sum + item.value, 0);
                    const percentage = total > 0 ? ((params.value / total * 100).toFixed(2)) : '0.00';
                    return `${percentage}%`;
                }, 
                color: '#fff', 
                position: 'outside' 
            }
        }]
    });
}
```

**关键要点：**
1. **初始化时保持简单配置**：不要在初始化时引用不存在的数据
2. **数据更新时设置完整配置**：通过闭包访问当前的 `pieData` 数组
3. **利用 ECharts 的增量更新机制**：`setOption` 会合并配置，不会覆盖未提及的选项

## 核心原则
**保证数据一致性：**
1. 所有显示的数值都先 `Math.round()` 四舍五入
2. 所有百分比计算都基于四舍五入后的值
3. 表格和图表使用完全相同的计算逻辑

**图表配置原则：**
1. 初始化时保持配置简单，避免引用不存在的数据
2. 在数据更新时设置完整的 formatter 配置
3. 利用闭包确保 formatter 能访问到正确的数据数组

**注意：** 百分比总和可能不是精确的100.00%，可能是99.99%或100.01%，这是正常的四舍五入误差，可以接受。
