// ========================================================================
// 【重要：通用数据处理规则】
// ========================================================================
// 
// 所有总统计页面必须遵循的核心规则：
// 
// 1. SQL查询规则：
//    ✅ 允许：SUM(CASE WHEN MONTH(date)=1 THEN sum_elec...) - 汇总每天数据得到月度值
//    ❌ 禁止：SUM(sum_elec) as sum_elec - 不要在SQL中直接累加年度合计
//    原因：年度合计需要先四舍五入每月值，再累加
//
// 2. JavaScript数据处理规则：
//    ✅ 正确流程：
//       a) SQL查询返回12个月的数据（month1_elec, month2_elec, ..., month12_elec）
//       b) JavaScript中为每行数据计算 sum_elec：
//          row.sum_elec = 0;
//          for (let m = 1; m <= 12; m++) {
//              const monthVal = row[`month${m}_elec`] || 0;
//              if (monthVal > 0) {
//                  row.sum_elec += Math.round(monthVal);  // 先四舍五入，再累加
//              }
//          }
//       c) 显示时所有电量值都使用 Math.round() 四舍五入
//       d) 百分比计算使用四舍五入后的值
//
// 3. 为什么要先四舍五入再累加？
//    示例：假设3个月的电量分别是 100.4, 200.6, 300.3
//    - 错误方式（先累加后四舍五入）：100.4 + 200.6 + 300.3 = 601.3 → 601
//    - 正确方式（先四舍五入再累加）：100 + 201 + 300 = 601
//    虽然这个例子结果相同，但在更多数据时，先四舍五入再累加能确保：
//    - 表格中显示的月度值与年度合计值一致
//    - 各单元的年度合计之和 = 总合计值
//
// 4. 百分比计算规则：
//    percentage = Math.round(roundedValue * 10000 / roundedTotal) / 100
//    说明：先放大10000倍，四舍五入，再缩小100倍，确保保留2位小数
//
// 5. 图表数据处理：
//    - 饼图数据：使用四舍五入后的值 Math.round(value)
//    - 饼图百分比：手动计算，不使用 ECharts 自动计算
//    - 曲线图数据：可以使用原始值（因为需要展示趋势）
//
// ========================================================================
