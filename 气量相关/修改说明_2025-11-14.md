# 气量统计画面修改说明

**修改日期：** 2025-11-14  
**修改文件：**
1. `js/utils/energy-config-v2.js` - 配置文件

## 一、修改概述

### 原有结构
- 表头分为**工厂本部**（包含工厂总用气量、设备用气量、辅助用气量）和其他部门
- 采用复杂的两层表头结构
- 包含8个部门分类（工厂、设备、辅助、动力设施课、开发本部、经营企划部、品质本部、营业本部）

### 新结构
- **平铺显示6个车间/部门**，没有工厂本部的分组
- 采用简单的两层表头结构（车间名称 + 用气量/占比）
- 新的分类：开发本部、新成品车间、机加工车间、组装车间、壳体车间、电机车间

---

## 二、新的分组计算规则

所有计算规则在配置文件中定义，支持**加法和减法运算**：

| 分类 | 计算公式 | NO编号说明 |
|------|---------|-----------|
| **开发本部** | NO.3 | 直接统计NO.3 |
| **新成品车间** | NO.2 - NO.3 | NO.2的气量减去NO.3的气量 |
| **机加工车间** | NO.4 - NO.5 | NO.4的气量减去NO.5的气量 |
| **组装车间** | NO.6 + NO.8 | NO.6和NO.8的气量相加 |
| **壳体车间** | NO.7 - NO.9 - NO.8 | NO.7的气量减去NO.9和NO.8的气量 |
| **电机车间** | NO.5 + NO.9 | NO.5和NO.9的气量相加 |
| **公司总用气量** | NO.2-9 的总和 | 全厂所有气表的总和 |

## 五、SQL查询逻辑

配置文件中的 `generateSQLCaseClauses()` 方法会自动生成SQL查询：

```sql
SELECT sub.month,
       SUM(CASE WHEN sub.no IN (3) THEN sub.total_gas ELSE 0 END) AS dev_dept_gas,
       (SUM(CASE WHEN sub.no IN (2) THEN sub.total_gas ELSE 0 END) - 
        SUM(CASE WHEN sub.no IN (3) THEN sub.total_gas ELSE 0 END)) AS new_product_gas,
       (SUM(CASE WHEN sub.no IN (4) THEN sub.total_gas ELSE 0 END) - 
        SUM(CASE WHEN sub.no IN (5) THEN sub.total_gas ELSE 0 END)) AS machining_gas,
       SUM(CASE WHEN sub.no IN (6,8) THEN sub.total_gas ELSE 0 END) AS assembly_gas,
       (SUM(CASE WHEN sub.no IN (7) THEN sub.total_gas ELSE 0 END) - 
        SUM(CASE WHEN sub.no IN (9,8) THEN sub.total_gas ELSE 0 END)) AS shell_gas,
       SUM(CASE WHEN sub.no IN (5,9) THEN sub.total_gas ELSE 0 END) AS motor_gas,
       SUM(CASE WHEN sub.no IN (2,3,4,5,6,7,8,9) THEN sub.total_gas ELSE 0 END) AS total_gas
FROM (
    SELECT no, MONTH(`date`) AS month, SUM(sum_FLOW_total) AS total_gas
    FROM sum_gas_flow_total_by_day
    WHERE YEAR(`date`) = [年份]
    GROUP BY no, MONTH(`date`)
) AS sub
GROUP BY sub.month
ORDER BY sub.month
```

## 七、注意事项

### 1. 减法运算可能的问题
某些车间使用减法计算，理论上可能出现**负值**：
- 新成品车间 = NO.2 - NO.3（如果NO.3 > NO.2，结果为负）
- 机加工车间 = NO.4 - NO.5（如果NO.5 > NO.4，结果为负）
- 壳体车间 = NO.7 - NO.9 - NO.8（如果NO.9+NO.8 > NO.7，结果为负）

**建议**：在实际使用中观察是否出现负值，如果出现，需要确认NO编号配置是否正确。

### 2. NO编号说明
- NO编号不带 `#` 符号
- NO编号对应数据库中的气表编号
- 目前使用NO: 2, 3, 4, 5, 6, 7, 8, 9（共8个气表）

### 3. 配置灵活性
如果将来需要修改计算规则，只需修改 `energy-config-v2.js` 中的 `REGIONAL_GAS_STATS.categories` 配置，无需修改HTML代码。
